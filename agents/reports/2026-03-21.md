## [리드 전략 업데이트] 2026-03-21 — 대표님 논의 확정사항

오늘 대표님과의 논의에서 확정된 전략 결정사항 기록.

### 1. 서버 리전
- **결정:** 서울 유지 (현행 유지)
- **이유:** Phase 1 트래픽 소스는 100% 한국. 이전은 호주 유저 100명+ 시점에 검토.
- **장기 방향:** Supabase Pro 전환 시 시드니 Primary + 서울 Read Replica 구조

### 2. SNS 마케팅 전략 (재정립)
- **채널 운영:** 익명, 얼굴 노출 없음
- **콘텐츠 소스:** 대표님 직접 유학 경험 + 실제 경험자 친구 네트워크 기반 (인터넷 재가공 금지)
- **콘텐츠 믹스:** 유학 리얼 후기 50% / 유학 준비 정보 25% / 밈 15% / Yuhu 소개 10%
- **포지셔닝:** "호주 유학의 어두운 부분을 밝게 보여주는 채널"
- **SNS → 플랫폼 연결:** "유학원 추천" 댓글/DM 니즈를 트리거로 Yuhu 연결
- **CTA 시점:** 플랫폼 리뷰 30개 확보 후 Yuhu 링크 노출 시작

### 3. 오픈 전 리뷰 수집 전략
- **방법:** 구글폼 → 구글 시트 → CSV → Supabase 적재
- **구글폼 설계:** 기존 폼 폐기, DB 구조 기반으로 신규 설계
  - 폼 A — 상담 후기 (12개 항목)
  - 폼 B — 등록/사후관리 후기 (18개 항목)
- **SNS 끝단 CTA:** "유학원 후기 남겨주세요" → 구글폼 링크 (Yuhu 링크 아님)

### 4. 학교 데이터 확장 방향
- **추가 타입:** 어학원(language) / TAFE / 사립 RTO / 파운데이션
- **수집 범위:** 기본 정보 수준 (기관명/도시/웹사이트/주요 코스/학비 범위)
- **목적:** 리뷰 시스템 아님. 참고 정보 제공 + 학교생활 커뮤니티의 배경 데이터.
- **우선순위:** 어학원 → TAFE → 사립 RTO·파운데이션 (여유 있을 때)

### 5. 서비스 구조 확정
```
유후 플랫폼
├── 유학원 리뷰 (메인) — 한국에서 유학 준비 중인 사람 타겟
└── 학교생활 게시판 (부가) — 호주에 이미 와 있는 사람 타겟, 커뮤니티
```

### 6. 에이전트 운영 체계 정비
- 호칭/어투 통일: 대표님 호칭 + 존댓말 전 에이전트 적용 완료
- 세션 시작 인사 프로토콜 전 에이전트 적용 완료
