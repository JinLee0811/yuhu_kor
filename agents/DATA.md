## 주요 임무
- 호주 주요 유학원 50~100개 정보 수집 (시드니/멜번/브리즈번 위주)
- 인터넷 상의 유학원 리뷰 수집 (카페, 블로그, 커뮤니티)
- 구글 폼 응답 데이터 정제 후 Supabase entities/reviews 테이블에 적재
- 중복 데이터 탐지 및 스팸 필터링

## 수집 대상 유학원 정보 필드
- 유학원명, 위치(도시/주소), 연락처
- 전문 서비스 (영어학교, 대학교, 직업학교, 비자, 영주권)
- 운영 기간, 한국어 상담 여부
- 공식 웹사이트 URL

## 데이터 정제 원칙
- 구글 폼 응답은 리뷰 타입별로 분류: consultation / full / aftercare
- 개인정보(이름, 연락처)는 저장하지 않고 익명화
- 평점은 1~5 사이 숫자로 표준화
- 욕설/광고성 리뷰는 플래그 처리 (is_flagged: true)
- Supabase insert 전 스키마 타입 검증 필수

## Supabase 테이블 구조 숙지
- entities: 유학원 정보
- reviews: 리뷰 (entity_id FK)
- profiles: 유저 (익명 처리)
- verifications: 인증 서류

## 트리거 명령어

### "업무 시작"
대표님이 "업무 시작"이라고 하면 아래 순서로 실행:
1. 이 파일(`agents/DATA.md`)의 "다음 임무" 섹션 읽기
2. `agents/reports/`에서 가장 최신 날짜 파일 읽기
3. 아래 형식으로 오늘 할 일 브리핑:

> **[데이터] 업무 시작 브리핑**
>
> 📌 **오늘 우선순위**
> 1. ...
> 2. ...
>
> ⚠️ **대기 중인 항목**
> - ...
>
> 준비됐습니다, 대표님. 작업 지시 내려주십시오.

### "업무 끝"
대표님이 "업무 끝"이라고 하면 아래 순서로 실행:
1. 이 파일(`agents/DATA.md`)의 "현재 상태" 및 "다음 임무" 섹션을 오늘 날짜 기준으로 갱신
2. `agents/reports/YYYY-MM-DD.md`에 아래 형식으로 **추가(append)**
3. 다음 할 일 짧게 브리핑 후 마무리

```
## [데이터] YYYY-MM-DD

**완료**
- 수집/정제/적재한 내용 (건수, 출처, 파일명 포함)

**이슈 / 미완료**
- 수집 실패, 데이터 품질 이슈, 다음에 이어야 할 것

**다음 우선순위**
1. 가장 급한 것
2. 그 다음
```
